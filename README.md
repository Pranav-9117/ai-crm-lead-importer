# AI-Powered CRM Lead Importer (v2.0)

A robust, type-safe fullstack solution for ingesting, AI-validating, normalizing, and importing arbitrary CRM lead CSV files into a standardized CRM schema.

## Architecture & Foundation (`SPEC-0001`)

This repository follows a clean two-application architecture without monorepo workspace coupling (`Turborepo`, `Nx`):

```text
ai-crm-lead-importer/
├── frontend/                     # Next.js 14 Client Application (Stitch MCP UI + API Service layer)
│   ├── src/
│   │   ├── app/                  # (generated by Antigravity from Stitch) App Router Pages
│   │   ├── components/           # (generated by Antigravity from Stitch) UI Components
│   │   ├── services/             # (developer-owned) API Client Wrappers & Networking
│   │   └── types/                # (developer-owned) Matching Frontend API Contract Types
├── backend/                      # Node.js + Express + TypeScript API Server
│   ├── src/
│   │   ├── controllers/          # Express Route Controllers
│   │   ├── middlewares/          # Express Middlewares (`error`, `validation`)
│   │   ├── services/             # Domain Orchestrators
│   │   ├── validators/           # Zod Runtime Schemas
│   │   ├── types/                # Canonical Backend API Contract Types (`LeadDTO`, `ApiResponse<T>`)
│   │   └── utils/errors/         # Functional Error Handling (`AppError`)
```

## Getting Started

### 1. Backend Setup

```bash
cd backend
npm install
npm run dev
```
The backend API runs on `http://localhost:3001` (`PORT=3001` by default). Check health endpoint via: `GET http://localhost:3001/api/health`.

### 2. Frontend Setup

```bash
cd frontend
npm install
npm run dev
```
The Next.js client runs on `http://localhost:3000`.

## API Contract & Type Mirroring

The backend (`backend/src/types/`) is the canonical source of truth for the API contract (`ApiResponse<T>`, `LeadDTO`, `CSVRow`, `CRMStatusEnum`, `DataSourceEnum`). The frontend (`frontend/src/types/`) maintains an exact mirror for compile-time safety without shared build step dependencies.
